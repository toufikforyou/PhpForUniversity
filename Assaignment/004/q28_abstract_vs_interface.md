# Question 28: Abstract Class vs Interface

| Feature | Abstract Class | Interface |
| :--- | :--- | :--- |
| **Methods** | Can have both abstract (no body) and concrete (with body) methods. | Can only have abstract methods (public). |
| **Properties** | Can have properties (variables). | Cannot have properties (only constants). |
| **Inheritance** | A class can extend only one abstract class. | A class can implement multiple interfaces. |
| **Access Modifiers** | Methods can be public, protected, or private. | All methods must be public. |
| **Keyword** | Defined using `abstract class`. | Defined using `interface`. |
